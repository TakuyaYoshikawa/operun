## 1. 認証・マルチテナント

### 概要
JWT Bearer トークンによる認証。テナント（会社）ごとにデータを完全分離。

### 仕様
- ログイン：メールアドレス＋パスワード → JWT トークン発行
- トークン有効期限切れ時：自動ログアウト＆リロード（401インターセプター）
- 全APIはテナントIDでフィルタリング（他社データへのアクセス不可）
- ユーザーはテナントに紐づく（将来：複数ユーザー対応）

### データモデル
| テーブル | 主なカラム |
|---------|-----------|
| tenants | id, name, plan |
| users | id, tenant_id, email, hashed_password, name |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/auth/login | ログイン（フォームデータ） |
| GET | /api/auth/me | 現在のユーザー情報 |

## 2. 設備マスタ管理

### 概要
加工設備の登録・管理。設備グループによる自動割り付けと外注設備の管理をサポート。

### 仕様
- 設備コード（テナント内一意）、名称、稼働時間、段取り時間を管理
- **設備グループ（machine_type）**：同じグループの設備はスケジューリング時に自動選択（最も早く空く設備を採用）
- **稼働停止フラグ（is_active）**：停止中の設備はスケジューリング対象外
- **外注設備フラグ（is_outsource）**：外注先名・標準リードタイムを設定可能
- **設備グループのオートコンプリート**：既存グループ名を入力中に候補表示
- **表示順の並び替え**：ドラッグ＆ドロップで設備の表示順を変更可能
- **削除保護**：工程実績に使用中の設備は削除不可（エラーメッセージ表示）
- **メンテナンス枠**：設備ごとにメンテナンス期間を登録（スケジューリングで非稼働扱い）

### データモデル（machines）
| カラム | 型 | 説明 |
|--------|-----|------|
| name | string | 設備名（例：旋盤1号機） |
| code | string | 設備コード（例：M001） |
| daily_capacity_hours | float | 1日の稼働時間（デフォルト：テナント設定値） |
| setup_time_minutes | float | 段取り時間（分）※スケジューリングに加算 |
| machine_type | string | 設備グループ名（例：旋盤） |
| is_active | bool | 稼働中フラグ |
| is_outsource | bool | 外注設備フラグ |
| outsource_supplier | string | 外注先名 |
| outsource_lead_days | int | 標準リードタイム（日） |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/machines | 設備一覧 |
| POST | /api/machines | 設備登録 |
| PUT | /api/machines/{id} | 設備更新 |
| DELETE | /api/machines/{id} | 設備削除（使用中は400エラー） |
| PUT | /api/machines/reorder | 設備の表示順変更 |
| GET | /api/machines/{id}/maintenance | メンテナンス一覧 |
| POST | /api/machines/{id}/maintenance | メンテナンス登録 |
| DELETE | /api/machines/{id}/maintenance/{m_id} | メンテナンス削除 |

> **注記**：工程マスタ（process）はUIから廃止。設備グループ（machine_type）と機能が重複するため削除した。APIエンドポイント（/api/machines/processes/）はバックエンドに残存。

## 5. 品番テンプレート

### 概要
品番ごとの標準工程（簡易BOM）を登録。受注登録時の工程自動展開に使用。

### 仕様
- 品番コード・品名に対して、複数の標準工程（順序・設備・作業時間）を登録
- **新規受注時のテンプレート選択**：受注登録モーダル（ガントチャート・受注管理どちらからでも）でテンプレートを選択すると品名・品番を自動入力し、受注登録後に工程を一括自動追加
- マスタ管理画面の「品番テンプレート」タブから手動登録・削除が可能

### テンプレート工程フィールド
| フィールド | 説明 |
|-----------|------|
| sequence | 工程順序 |
| machine_id | 設備ID（グループ自動選択） |
| hours_per_unit | 作業時間（h） |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/product-templates | テンプレート一覧 |

| POST | /api/product-templates | テンプレート登録 |

## 6. 受注管理

### 概要
製造受注の登録・管理。工程展開とスケジューリングのトリガー。

### 仕様
- 受注番号（テナント内一意）、品名・品番、数量、納期、優先度、顧客を管理
- 優先度：1=特急、2=高、3=通常
- ステータス：pending（未着手）/ in_progress（進行中）/ done（完了）
- 受注削除時は紐づく工程も一括削除（cascade）
- CSVインポート対応（受注・工程のバルク登録）

### 画面仕様（受注管理ページ）
- **一覧表示**：受注番号・品名・品番・数量・納期・優先度・ステータス・取引先
- **行クリックで詳細展開**：選択した受注の編集フォームと工程一覧をインライン表示（別ページ遷移なし）
- **工程一覧**：各工程の設備・時間・ステータス・計画日時をインラインで表示・編集
- **「＋ 新規登録」ボタン**：ページ上部のフォームから新規受注を登録

### 画面仕様（受注登録・編集モーダル）
ガントチャートの「＋ 新規受注」ボタン、またはガントバークリックから開く。

- **新規作成モード**
  - 品番テンプレート選択（任意）→ 品名・品番を自動入力、工程プレビュー表示
  - 登録後、テンプレートの工程を自動追加してそのまま編集モードに遷移
  - 取引先の「＋ 新規登録」でインライン顧客登録
- **編集モード**
  - 受注情報（品名・品番・数量・納期・優先度・ステータス・取引先・備考）の編集
  - 工程一覧の表示・追加・編集・削除
  - 各工程に対して着手・完了・スケジュールロック操作が可能
  - 受注削除ボタン

### 工程（Operation）フィールド
| フィールド | 説明 |
|-----------|------|
| machine_id | 設備ID |
| machine_locked | true=設備固定、false=グループ自動選択（デフォルト: false） |
| duration_hours | 作業時間（h） |
| wait_hours_after | 完了後待機時間（h）：冷却・乾燥等 |
| not_before_date | 開始不可日：この日以前に開始しない |
| is_urgent | 特急フラグ |
| op_status | not_started / in_progress / done / on_hold |
| schedule_locked | スケジュール固定（着手中・完了は自動でtrue） |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/orders | 受注一覧（ステータス・品名フィルタ可） |
| POST | /api/orders | 受注登録 |
| GET | /api/orders/{id} | 受注詳細（工程含む） |
| PUT | /api/orders/{id} | 受注更新 |
| DELETE | /api/orders/{id} | 受注削除（工程も削除） |
| POST | /api/orders/{id}/operations | 工程追加 |
| PUT | /api/orders/{id}/operations/{op_id} | 工程更新 |
| DELETE | /api/orders/{id}/operations/{op_id} | 工程削除 |

## 7. スケジューリングエンジン

### 概要
受注工程を設備に自動割り付けしてスケジュールを生成。

### アルゴリズム
- **EDD（Earliest Due Date）フォワードスケジューリング**（フォールバック）
  - ソート順：特急フラグ → 優先度 → 納期 → 受注ID → 工程順序
  - 設備の空き時刻から順に割り付け（日またぎ・休日・非稼働時間考慮）
- **OR-Tools 最適化**（`/schedule/run` のデフォルト）
  - Google OR-Tools の CP-SAT ソルバーによる数理最適化
  - OR-Toolsが未インストールの場合はEDDにフォールバック

### 主要仕様
- **段取り時間の考慮**：`duration_hours + setup_time_minutes / 60` で実効時間を計算
- **設備グループ対応**：`machine_locked=false` の工程は同一グループ内で最も早く空く設備を自動選択
- **設備固定**：`machine_locked=true` の場合は指定設備に固定
- **工程順序の依存関係**：同一受注の工程は前工程終了後に次工程を開始
- **完了後待機時間**：`wait_hours_after > 0` の場合、次工程の開始をその時間分遅延
- **開始不可日**：`not_before_date` が指定されている場合、その日以前に開始しない
- **稼働カレンダー対応**：休日・非稼働時間をスキップ
- **スケジュールロック**：`schedule_locked=true` の工程は再スケジューリングで移動しない（着手中・完了は自動ロック）

### 下書きスケジュール機能
スケジュールは現行（確定済み）と下書きの2段階で管理。

| 操作 | 説明 |
|------|------|
| 下書き作成 | 現行スケジュールをそのまま下書きカラムにコピー |
| スケジュール最適化 | OR-Toolsで最適化した結果を下書きカラムに書き込み |
| 下書き編集 | 個別工程の日時・設備を手動変更 |
| 確定 | 下書き → 現行スケジュールに反映、下書きを削除 |
| 破棄 | 下書きを削除（現行スケジュールはそのまま） |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/schedule/run | スケジュール最適化（下書きに保存） |
| POST | /api/schedule/create-draft | 現行スケジュールを下書きにコピー |
| PATCH | /api/schedule/draft/{op_id} | 下書き工程の手動編集 |
| POST | /api/schedule/commit | 下書きを確定 |
| POST | /api/schedule/discard | 下書きを破棄 |
| GET | /api/schedule/gantt | ガントチャート用データ（draft パラメータで切替） |
| POST | /api/schedule/toggle-lock/{op_id} | スケジュールロック切替 |
| POST | /api/schedule/simulate/delivery | 納期シミュレーション |

## 8. ガントチャート

### 概要
スケジュールを設備行×時間軸で可視化。現行・下書きの切替と手動編集をサポート。

### 表示仕様
- **X軸**：日単位（デフォルト）または時間単位（稼働開始時刻〜終業時刻）を切替可能
- **Y軸**：設備ごとに1行
- **バーの色**：受注ごとに異なる色（12色パレットから循環）
- **バーのラベル**：品名のみ表示
- **特急マーカー**：`!` アイコン表示
- **納期超過**：赤い枠線で表示
- **土日列**：土曜=青背景、日曜=赤背景
- **ホバーツールチップ**：品名・受注番号・設備・開始/終了・納期・優先度を表示

### バークリック
- **現行モード**：バーをクリックすると受注詳細・編集モーダル（OrderModal）が開く
- **下書きモード**：バーをクリックすると工程の日時・設備を手動変更するモーダルが開く

### 新規受注
- ガントチャート上部の「＋ 新規受注」ボタンから受注登録モーダルを開く
- 品番テンプレートの選択→工程自動展開に対応

### 下書きモード
- 下書きがある場合、現行/下書きの切替タブを表示
- 下書きモードではバーに破線アウトラインと ✎ アイコン
- バークリックで日時・設備を変更するモーダルを表示
- 変更は即座にDBに保存（中断しても保持）

### ガントAIアシスタントパネル
ガントチャート下部に常時表示されるAIチャットパネル。

- スケジュール状況の確認・調整を自然言語で指示
- エージェントモードで受注更新・スケジュール再実行などを直接操作
- 書き込み操作はユーザー確認後に実行

### 設備負荷表示
- 設備ごとの稼働率・負荷時間を可視化

## 9. 工程実績入力（現場向け）

### 概要
スマートフォン等での現場からの工程実績入力を想定したUI。

### 仕様
- **対象工程**：当日（または直近の稼働日）の工程を自動表示
- **設備フィルター**：担当設備で絞り込み
- **日付切替**：今日 / 日付指定で切替
- **自動更新**：30秒ごとに最新情報を取得
- **ステータス管理**：not_started → in_progress → done / on_hold
- **経過時間表示**：進行中の工程は開始からの経過時間をリアルタイム表示
- **完了時の自動処理**：全工程完了で受注ステータスを自動的に done に変更

### ステータス遷移
| 操作 | 遷移前 | 遷移後 |
|------|--------|--------|
| 開始 | not_started | in_progress |
| 完了 | in_progress | done（実績時間を自動計算） |
| 保留 | in_progress | on_hold（理由を記録） |

### API
| メソッド | パス | 説明 |
|---------|------|------|
| GET | /api/operations/today | 本日の工程一覧 |
| GET | /api/operations | 工程一覧（フィルタ可） |
| POST | /api/operations/{id}/start | 開始 |
| POST | /api/operations/{id}/complete | 完了 |
| POST | /api/operations/{id}/hold | 保留 |

## 10. 納期シミュレーター

### 概要
新規受注を仮想的に差し込んだ場合の完成予定日と既存受注への影響を即座に試算。

### 仕様
- 入力：品名・設備グループ・作業時間・希望納期・優先度・特急フラグ
- 出力：
  - 完成予定日（例：「2026年4月8日（水）」）
  - 今日から完成までの営業日数
  - 納期内完成の可否
  - 影響を受ける既存受注の件数・受注番号
- AI による自然言語での結果説明（電話口での顧客説明文として利用可能）
- シミュレーション後にそのまま受注登録が可能

### API
| メソッド | パス | 説明 |
|---------|------|------|
| POST | /api/schedule/simulate/delivery | 納期シミュレーション |
| POST | /api/ai/explain-simulation | 結果をAIが日本語文章化 |

## 11. AIアシスタント

### 概要
Claude（Anthropic）を活用したAIアシスタント。複数の場所・モードで利用可能。

### 利用場所

| 場所 | 機能 |
|------|------|
| ガントチャート下部パネル | スケジュール確認・調整をエージェントモードで操作 |
| AIアシスタント画面 | チャット・受注入力解析・エージェント全機能 |
| マスタ管理画面 | 各マスタタブに対応したAIパネルで音声入力的に登録 |

### モード

#### チャットモード
- スケジュール状況の質問に日本語で回答
- ガントサマリーをコンテキストとして自動付与
- 使用モデル：claude-haiku-4-5

#### 受注登録モード（自然言語入力）
- 「部品A 50個 来月末まで」のような自然言語テキストから受注フォームを自動生成
- 複数受注の一括解析に対応
- 不足フィールドを missing_fields として返却

#### エージェントモード
- ツールを呼び出してシステムを直接操作
- **確認フロー**：書き込み操作は実行前にユーザーに内容を提示し、承諾後に実行
- 使用モデル：claude-sonnet-4-6

### エージェント利用可能ツール

| ツール | 操作 |
|--------|------|
| search_orders | 受注を検索・確認 |
| update_order | 受注の納期・優先度・数量・ステータス・備考を変更 |
| create_order | 新規受注を登録 |
| search_materials | 材料・在庫を検索 |
| receive_stock | 材料の入庫登録 |
| issue_stock | 材料の払出登録 |
| create_purchase_order | 材料の発注を登録 |
| search_customers | 顧客を検索 |
| create_customer | 顧客を登録 |
| create_machine | 設備を登録 |
| get_schedule_summary | スケジュール状況サマリーを取得 |
| run_schedule | スケジュールを再実行 |

### 操作例
```
ユーザー：「ORD-007の納期を5月25日に変更してください」
AI：「ORD-007（スリーブ部品G）の納期を4/8 → 5/25に変更します。よろしいですか？」
ユーザー：「はい」
AI：「納期を変更しました。スケジュールを再実行しますか？」
```

### API
| メソッド | パス | 説明 |
|---------|------|------|

## 14. 安全性・エラーハンドリング

### データ整合性
- **マスター削除保護**：使用中の設備・顧客は削除不可（400エラーを返し、UIにメッセージ表示）
- **受注削除時のカスケード**：受注削除時は紐づく工程も一括削除

### フロントエンド安全性
- **ErrorBoundary**：コンポーネントのレンダリングエラーをキャッチし、再読み込みボタンを表示
- **401インターセプター**：トークン期限切れを検知し、自動ログアウト＆リロード
- **throwOnError: false**：TanStack Queryのクエリエラーが ErrorBoundary を誤発火させない設定

### 認証
- リクエストインターセプターでlocalStorageからトークンを毎回読み込み（リロード時の認証漏れ防止）
