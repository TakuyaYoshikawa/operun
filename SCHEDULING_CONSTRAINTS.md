# Operun スケジューリング制約仕様

実装済みのスケジューリングエンジン（`backend/app/scheduler/engine.py`）に組み込まれている制約・機能の一覧。

---

## アルゴリズム概要

**EDD（Earliest Due Date）フォワードスケジューリング**

- 納期が早い受注を優先して設備に順次割り付ける
- 現時刻から先向きにスケジュールを組む
- Phase 3 でOR-Tools CP-SATソルバーに差し替え予定

---

## 優先順位制御

### 特急フラグ（最優先）
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.is_urgent` |
| 動作 | 特急フラグのある工程を全て最優先でスケジュール |
| ソート順 | 特急 → 優先度 → 納期 → 受注ID → 工程番号 |

### 優先度
| 値 | 意味 |
|----|------|
| 1 | 高（特急に次ぐ優先） |
| 2 | 中 |
| 3 | 通常（デフォルト） |

## カレンダー制約

### 稼働時間
| 項目 | 詳細 |
|------|------|
| 設定場所 | `TenantSettings.work_start_hour` / `work_hours_per_day` |
| 設備個別設定 | `Machine.work_start_hour`（テナント設定を上書き） |
| 動作 | 稼働時間外は自動的に翌稼働時間の開始まで待機 |

### 休日・全休日
| 項目 | 詳細 |
|------|------|
| テーブル | `CalendarHoliday`（`working_hours = 0`） |
| 動作 | その日は稼働ゼロ扱い。翌営業日の開始時刻にスキップ |
| 日曜 | 常に非稼働（ハードコード） |

### 土曜休み
| 項目 | 詳細 |
|------|------|
| 設定 | `TenantSettings.saturday_off` |
| 動作 | `true` の場合、土曜を全休日と同様に扱う |

### 短縮稼働日（半日・特別稼働）
| 項目 | 詳細 |
|------|------|
| テーブル | `CalendarHoliday`（`0 < working_hours < 通常稼働時間`） |
| 動作 | その日の稼働時間上限を `working_hours` に制限 |
| 例 | お盆期間の半日稼働（4時間）など |

## 設備制約

### 設備グループ（代替設備の自動選択）
| 項目 | 詳細 |
|------|------|
| 設定 | `Machine.machine_type`（同一文字列でグループ化） |
| 動作 | 同グループの設備から最も早く終わるものを自動選択 |
| 例 | 旋盤1号機・旋盤2号機 → 空いている方を自動割り当て |
| 無効化 | `Operation.machine_locked = true` で固定設備を強制 |

### 設備固定（機械ロック）
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.machine_locked` |
| 動作 | `true` の場合、グループ内自動選択を行わず指定設備に固定 |

### バッチ処理（同時並行）
| 項目 | 詳細 |
|------|------|
| 設定 | `Machine.batch_capacity` |
| 動作 | 同一設備で `batch_capacity` 件まで同時処理可能 |
| 例 | 焼入れ炉（複数ワークを同時投入）など |
| 実装 | スロット管理方式（最も早く空くスロットに割り当て） |

### 段取り時間
| 項目 | 詳細 |
|------|------|
| 設定 | `Machine.setup_time_minutes` |
| 動作 | 実加工時間に段取り時間を加算してスケジュール |
| 例 | 治具交換30分 → 実加工8h → 合計8.5hで計算 |

### メンテナンス枠
| 項目 | 詳細 |
|------|------|
| テーブル | `MachineMaintenance`（開始・終了日時） |
| 動作 | メンテナンス期間中はその設備を使用不可。終了後に再開 |
| 例 | 定期点検・修理・清掃など |

### 外注設備
| 項目 | 詳細 |
|------|------|
| フィールド | `Machine.is_outsource = true` |
| 設定 | `Machine.outsource_lead_days`（標準リードタイム日数） |
| 動作 | カレンダー無視でリードタイム分の経過時間として計算 |
| 例 | 外注メッキ処理：3営業日 → 72時間を単純加算 |

## 工程間制約

### 工程順序依存
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.sequence` |
| 動作 | 同一受注の工程は `sequence` 順に完了してから次工程を開始 |
| 保証 | 前工程終了時刻 ≤ 次工程開始時刻 |

### 工程間待機時間
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.wait_hours_after` |
| 動作 | 工程完了後、指定時間が経過するまで次工程を開始しない |
| 例 | 塗装後24時間乾燥、焼入れ後冷却待ちなど |

### 材料調達待ち（開始不可日）
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.not_before_date` |
| 動作 | 指定日より前に工程を開始しない |
| 例 | 材料入荷予定日が4/20なら、それ以前に割り付けない |

## 手動調整・ロック

### スケジュールロック
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.schedule_locked` |
| 動作 | `true` の場合、再スケジュール時に日時・設備を上書きしない |
| 用途 | 確定済みの工程を再スケジュールの影響から保護 |
| 優先処理 | ロック済み工程を先に処理してスロットを占有させ、他工程を後から詰める |

### 下書きモード
| 項目 | 詳細 |
|------|------|
| フィールド | `Operation.draft_start` / `draft_end` / `draft_machine_id` |
| 動作 | スケジュール実行結果をまず下書きに保存、確認後に確定 |
| 手動編集 | ガントチャートからバーをクリックして日時・設備を変更可能 |

## 納期管理

### 納期超過検出
| 項目 | 詳細 |
|------|------|
| 判定 | `planned_end > due_date 17:00` |
| 出力 | `is_delayed`フラグ・`delay_days`（遅延日数） |
| UI | ガントチャートで赤枠表示・遅延サマリに一覧表示 |

### 納期シミュレーター（F-05）
| 項目 | 詳細 |
|------|------|
| エンドポイント | `POST /api/schedule/simulate/delivery` |
| 動作 | 新規受注を仮に差し込んだ場合の完了予定日と影響受注数を返す |

---

## 負荷グラフ

| 項目 | 詳細 |
|------|------|
| エンドポイント | `GET /api/schedule/load` |
| 表示 | 設備×日付の稼働率マトリクス（21日間） |
| 色分け | 超過：赤 / 高負荷(≥85%)：オレンジ / 中(≥50%)：青 / 低：薄青 |
