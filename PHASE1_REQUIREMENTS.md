# Operun Phase 1 機能要件書
## 金属加工・部品加工業向け軽量スケジューラ
### 対象期間：1〜6週目 / 目標：最初の1社に無料で使ってもらう

---

## 機能一覧（全6機能）

### 必須機能（なければ使い物にならない）

#### F-01 受注・工程マスタ登録
- 品番・工程名・標準作業時間・担当設備を登録
- CSVインポートで既存Excelデータをそのまま取り込める
- 設備マスタ：設備名・1日稼働時間・標準段取り時間を設定

#### F-02 ガントチャート自動生成
- 受注を入力するとEDDアルゴリズムで設備ごとに自動配置
- ドラッグ＆ドロップで手動調整可能
- 縦軸：設備、横軸：時間軸（日単位）

#### F-03 基本制約設定
- 稼働日・非稼働日（祝日・会社休日）の設定
- 設備ごとの1日稼働時間
- 設備ごとの固定段取り時間

#### F-04 納期アラート
- 納期を超過しそうな受注をガントチャート上で赤く表示
- ダッシュボードに「要注意受注一覧」を表示
- 遅延日数も合わせて表示

---

### 追加機能（Phase 1に組み込む・差別化の起点）

#### F-05 納期回答支援（新規追加）

**目的**
受注前に「いつ納品できるか」を営業担当者が即答できるようにする。
金属加工会社で最も頻繁に発生する「納期いつですか？」への回答を、
勘や経験ではなく計画データから根拠を持って答えられる。

**画面構成**
```
┌─────────────────────────────────────┐
│  📅 納期シミュレーター               │
│  ─────────────────────────────────  │
│  品番      [ ABC-001        ▼ ]     │
│  数量      [   50   ] 個            │
│  使用設備  [ 旋盤1号機      ▼ ]     │
│  優先度    [ ○ 通常  ● 特急 ]       │
│                                      │
│         [ 納期を確認する ]           │
│                                      │
│  ┌───────────────────────────────┐  │
│  │ ✅ 受注可能                     │  │
│  │                                 │  │
│  │ 完成予定  2026年4月8日（水）    │  │
│  │ 所要日数  5 営業日              │  │
│  │                                 │  │
│  │ ⚠️ 影響を受ける受注：1件        │  │
│  │   ORD-042 が1日後ろ倒しになります│  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

**動作仕様**
1. 品番・数量・設備・優先度を入力
2. 現在のスケジュールに差し込んだ場合の完成日を計算
3. 既存受注への影響（遅延が発生する受注）を合わせて表示
4. 特急指定の場合は最優先で割り込ませた結果を返す

**バックエンド実装**
既存の `simulate_insert()` をラップするだけ。追加コード約30行。

```python
@router.post("/simulate/delivery")
def check_delivery_date(payload: dict, db: Session = Depends(get_db)):
    result = simulate_insert(payload, db)
    completion = result["new_order_completion"]
    return {
        "feasible": completion is not None,
        "completion_date": format_date(completion),
        "business_days": calc_business_days(date.today(), completion),
        "affected_orders": result["delayed_order_numbers"],
        "affected_count": result["total_delayed_orders"],
    }
```

**営業現場での使い方**
- 顧客から電話で問い合わせがあった瞬間に画面を開いて確認
- 「今すぐ確認します」→ 30秒で回答 → 信頼感につながる
- 特急対応が必要な場合、既存顧客への影響を把握した上で判断できる

### 意図的に除外するもの（作りすぎない）

| 除外項目 | 理由 |
|---|---|
| 在庫管理 | 別システムとの競合になる・開発コスト大 |
| 受発注管理 | スコープ外・ERPの領域 |
| 原価計算 | 複雑すぎてPhase 1には不要 |
| 進捗実績入力 | Phase 2で対応 |
| モバイルアプリ | Webブラウザで十分 |
| 帳票印刷 | Phase 2（作業指示書）で対応 |

## デモシナリオ（最初の1社向け）

1. **現状再現**：「今Excelでやっている計画をそのまま入力してもらう」
2. **ガントチャート表示**：「これが今の計画です」
3. **割り込みデモ**：「特急が入ってきました。どうなりますか？」→ 即再計算
4. **納期回答デモ**：「今から電話があったとして、この受注いつ上がりますか？」→ 30秒で回答
5. **納期アラート**：「来週遅れそうな受注が2件あります」
